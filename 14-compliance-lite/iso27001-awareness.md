# ISO 27001 Awareness

ISO 27001 is the international standard for information security management. You don't need to be certified to benefit from it — but if you sell to enterprise, you'll probably be asked about it.

---

## What ISO 27001 Is

ISO 27001 defines an Information Security Management System (ISMS): a set of policies, processes, and controls for managing information security risks systematically.

Unlike SOC 2 (a US audit standard), ISO 27001 is internationally recognised. Many European enterprises and governments require it of their suppliers.

---

## Do You Need It?

| Situation | ISO 27001 Need |
|-----------|---------------|
| Consumer SaaS / B2C | No |
| Small B2B SaaS | No (but good practice) |
| Mid-market B2B SaaS with EU/UK enterprise customers | Likely to be asked |
| Government or regulated industry customers | Often required |
| Global enterprise software | Expected |

You'll know you need it when enterprise procurement asks: "Are you ISO 27001 certified?"

---

## The 114 Controls (Annex A)

ISO 27001 includes 114 controls across 14 categories:

1. Information security policies
2. Organisation of information security
3. Human resource security
4. Asset management
5. Access control
6. Cryptography
7. Physical and environmental security
8. Operations security
9. Communications security
10. System acquisition, development, and maintenance
11. Supplier relationships
12. Information security incident management
13. Business continuity management
14. Compliance

You don't implement all 114. You perform a risk assessment and implement controls relevant to your risks (with documented justification for exclusions).

## Practical Path to ISO 27001

### Timeline: 12–18 months for a startup

**Month 1–2: Scope and gap analysis**
- Define what's in scope (your product, infrastructure, team)
- Identify what you already do vs what's missing

**Month 3–6: Build the ISMS**
- Write security policies (access control, acceptable use, incident response)
- Perform and document your risk assessment
- Implement missing controls

**Month 7–12: Operate and evidence**
- Run the ISMS for 6 months minimum
- Collect evidence (access logs, training records, vendor assessments)
- Internal audit

**Month 12–18: Certification audit**
- Stage 1: Document review by certification body
- Stage 2: On-site/remote audit
- Certification issued (valid 3 years with annual surveillance audits)

## ISO 27001 vs SOC 2

| Factor | ISO 27001 | SOC 2 |
|--------|----------|-------|
| Geography | Global | US-centric |
| Governed by | ISO/IEC | AICPA |
| Type | Management system standard | Audit report |
| Frequency | 3-year cycle + annual audits | Annual |
| Output | Certificate | Report |
| Enterprise preference | EU/UK/Global | US |

If selling to US enterprise: SOC 2. If selling to EU/UK/global enterprise: ISO 27001 (or both).
