# Security Policy

## Reporting a Vulnerability

If you discover a vulnerability in OpenAuditor or its guidance, please report it responsibly.

**Do not** open a public GitHub issue for security vulnerabilities.

### How to Report

1. Email: security@baulin.tech
2. Include the following:
   - Description of the vulnerability
   - Affected section or file
   - Steps to reproduce (if applicable)
   - Potential impact
   - Your contact information for follow-up

### What Happens Next

1. We will acknowledge receipt within 48 hours
2. We will assess the severity
3. We will work on a fix and timeline
4. We will notify you before public disclosure
5. We will credit you in the fix (unless you prefer anonymity)

## Scope

This policy covers:
- **Documentation vulnerabilities** — Guidance that could lead to insecure implementations
- **Broken links or outdated advice** — Links to defunct tools, deprecated APIs
- **Missing critical guidance** — Major OWASP, MITRE, or CVE topics we should cover
- **Code examples with security flaws** — Examples that demonstrate insecurity

This policy does NOT cover:
- Requests for new features or sections (use GitHub issues instead)
- Questions about how to use the toolkit (use GitHub discussions)
- Vulnerabilities in tools we recommend (report to those projects directly)

## What We Promise

- We take all reported vulnerabilities seriously
- We will not publicly disclose your report without your consent
- We will credit you in our fixes
- We will prioritise critical guidance issues over minor ones

## Transparency

Once a fix is released, we will document:
- What the vulnerability was
- How it was fixed
- Who reported it (with permission)
- Any lessons learned

## Questions?

If you have questions about this policy, email security@baulin.tech.
